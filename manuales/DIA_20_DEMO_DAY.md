## Conceptos clave

El Demo Day es el cierre del curso. Cada alumno presenta su proyecto personal mostrando: el problema que resuelve, la arquitectura (entidades, capas, relaciones), una demo en vivo de los endpoints mas relevantes, y lo que aprendio durante el desarrollo.

La evaluacion entre pares usa criterios objetivos: funcionalidad (los endpoints funcionan), arquitectura (capas bien separadas), calidad del codigo (nombres claros, validaciones), deploy (Docker + CI/CD + Railway), y presentacion (claridad, tiempo, demo). Cada criterio se evalua de 1 a 5.

El curso cubre un camino completo: Java basico > OOP > Patrones > Maven > Hibernate > Spring Boot > Docker > CI/CD > Deploy. Cada dia construyo sobre el anterior. Lo que hiciste manualmente (DI, SQL, build) lo automatizaste con herramientas profesionales (Spring, JPA, Maven, Docker).

## Formato de presentacion

```
1. INTRODUCCION (1 min)
   - Nombre del proyecto y dominio elegido
   - Que problema resuelve

2. ARQUITECTURA (1 min)
   - Entidades y relaciones (mostrar diagrama ER)
   - Capas del proyecto

3. DEMO EN VIVO (3-4 min)
   - Mostrar Swagger UI
   - Crear un registro (POST)
   - Consultar registros (GET)
   - Mostrar una relacion entre entidades
   - Mostrar manejo de errores (404, 400)

4. CIERRE (1 min)
   - Tecnologias usadas
   - Que fue lo mas dificil / que aprendiste
   - Proximos pasos
```
